# サーバレスアーキテクチャ

サーバレスアーキテクチャとは実際にソフトウェアを実行するサーバがなくなるのではなく，クラウドによって提供されるサービスを活用することである．

以下のような 3 種類の考え方が主流である．

* 手元で管理するサーバをなくす．BaaS, PaaS
* ステートレスな Function 単位に処理を分割． FaaS
* クラウド上で提供されるコンポーネントを結びつけてシステムをつくり上げる．メッセージ駆動

# Linux カーネル観光ガイド

Linux 4.4 の新機能についての紹介

## I/O polling

NVMe SSD などのの登場によってレイテンシが低いデバイスを OS が意識する必要がでてきた．

現在の OS の I/O スタックはリクエストの完了を割り込みによって通知してもらうようにして作られている．
しかし，割り込み自体は高コストな処理であることに加えて，スケジューラが間に挟まることでアプリケーションから見たレイテンシが変動しやすくなるため，デバイスの性能を I/O システムが無駄にしてしまうことがある．

そこで I/O polling によって I/O 完了を待つという方式が追加された．

具体的には，preadv2/pwritev2 システムコールを使って `RWF_HIPRI` フラグをつけると I/O polling によって処理される．(Linux Kenrnel 4.6 以降)

Linux 4.10 では I/O polling の adaptive mode が追加され，しばらくスリープしてから polling を行うということも可能になっている．
(`queue/io_poll_delay` によって管理されている)

## LightNVM

### Background: FTL

SSD の特性を隠し，既存の I/O システムに適用するインターフェースを提供する．

* 論理アドレスと物理アドレスのマッピングを行う．
* GC
* 一部のセクタだけがk上に書かれないようにウェアレベリングを行う．

### 概要

NVMe をいい感じに扱おうとすると，FTL が邪魔になることがある．
LightNVM は FTL の機能を一部 OS 側に移すことで自由度高く NVMe を処理できる．
